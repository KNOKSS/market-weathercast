# 시장기상청 배포 메모 - 2026-06-27

## 이번 배포의 핵심

이번 버전은 기존 시장 날씨 앱에 `내일 시장예보 V2`와 자동 관측/정산 파이프라인을 포함한 배포본입니다.

중요 원칙:

- 오늘까지 빠진 날짜는 공식 기록으로 소급 입력하지 않습니다.
- 공식 shadow 예보 기록은 GitHub에 배포한 뒤 자동 워크플로우가 실제로 실행되는 시점부터 쌓습니다.
- 과거 데이터는 연구/검증용으로만 사용하고, 실제 운영 성과 기록과 섞지 않습니다.

## 포함된 주요 업데이트

### 1. 시장날씨 UI 정리

- `오늘 관측`과 `내일 예보` 모드를 분리했습니다.
- 개별 종목을 선택하면 상단 패널과 종목 카드가 즉시 반응하도록 개선했습니다.
- 모바일 PWA 화면에서 탭/선택 종목이 새로고침 후에도 유지되도록 구조를 정리했습니다.
- 상단 시간/새로고침 버튼은 오른쪽 정렬로 조정했습니다.
- 하단 불필요한 시장 하늘 카드는 제거하고, 전체시장 기준온도와 개별종목 날씨 역할을 분리했습니다.

### 2. 전체시장 기준온도 분리

- 사용자 추가 종목이 전체 시장 평균을 흔들지 않도록 분리했습니다.
- 전체시장 기준온도는 고정 시장 바스켓 기준으로 계산됩니다.
- 개별 종목 날씨는 선택 종목 자체의 체감온도/위험/모멘텀을 보여줍니다.

### 3. 종목별 내일 날씨 V2

- V2 모델 기반의 `다음 거래일 예상 활동 반경`을 앱에 표시합니다.
- 현재 방향 예측 모델이 아니라, 내일의 움직임 크기/변동폭 예보 모델입니다.
- 종목별 예보 카드에서 지원 종목을 추가/삭제할 수 있습니다.
- SOL은 기본 예보 목록에서 제거했습니다.
- 날씨 등급 옆에 아이콘을 추가했습니다.
- `폭풍` 같은 등급은 절대 퍼센트가 아니라 해당 자산의 최근 관측치 대비 상대적 강도를 뜻하도록 설명을 보강했습니다.

### 4. V2 모델/연구 시스템

- V2는 `log(True Range / prior close * 100)`을 타겟으로 쓰는 장 마감 후 다음 거래일 변동폭 예보 모델입니다.
- 방향성 예측은 아직 제공하지 않습니다.
- 검증 결과와 리포트 생성 스크립트가 포함되어 있습니다.
- 2025년 이후 모델 동결 상태의 예보-실제값 정산 리포트 구조를 추가했습니다.

### 5. 자동 예보/정산 파이프라인

포함된 자동화 파일:

- `.github/workflows/daily-market-forecast.yml`
- `scripts/shadow/run.py`
- `scripts/shadow/snapshot.ts`
- `scripts/v2_validation/run.py`
- `research-results/market-weather-shadow-v1`
- `research-results/market-weather-v2-validation-v1`
- `public/data/tomorrow-forecast.json`

GitHub Actions가 실행되면:

1. 완료된 일봉 데이터를 새로 가져옵니다.
2. 기존 예보 중 실제 결과가 나온 건을 정산합니다.
3. 다음 예보를 생성합니다.
4. 기준선/검증 리포트를 업데이트합니다.
5. 앱에서 쓰는 `public/data/tomorrow-forecast.json`을 갱신합니다.
6. 변경 사항을 GitHub에 자동 커밋합니다.

### 6. 뉴스/브리핑 파트

- 주의보 탭을 브리핑/뉴스룸 성격으로 개편했습니다.
- 핵심 시장 뉴스의 한국어 요약/표시 구조를 추가했습니다.

### 7. 안내 문구 강화

앱 하단에 다음 취지의 안내를 추가했습니다.

- 이 앱은 투자 조언이나 매수/매도 추천을 제공하지 않습니다.
- 시장 데이터를 기상 정보처럼 시각화한 보조 도구입니다.
- 모든 투자 판단과 책임은 사용자 본인에게 있습니다.
- 내일 시장예보는 과거 일봉 기반 통계 예보이며, 금리 발표·경제지표·실적·정책 변화·돌발 이벤트·천재지변 등 예보 이후의 외생 변수는 반영하지 않습니다.

## 사용자가 GitHub에 올릴 때 주의할 점

GitHub 웹사이트의 `Add files via upload`에는 ZIP 파일 자체를 올리는 것이 아니라, ZIP을 먼저 압축 해제한 뒤 안의 파일과 폴더를 업로드해야 합니다.

특히 아래 폴더가 빠지면 안 됩니다.

- `.github`
- `scripts`
- `research-results`
- `public`
- `src`
- `api`

반대로 아래 폴더는 업로드하지 않아도 됩니다.

- `node_modules`
- `dist`
- `.pnpm-store`

이 배포 ZIP은 위 불필요 폴더를 제외한 GitHub 업로드용 소스 패키지입니다.
