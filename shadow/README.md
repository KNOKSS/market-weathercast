# 전향적 shadow test v1

검증이 끝난 `presealed-candidate-v2`를 앱과 분리해 실제 장 마감 뒤 기록하는 연구 시스템입니다. 예측 원장과 다음 거래일 정산 원장을 각각 SHA-256 해시 체인으로 저장하며 기존 행을 수정하지 않습니다.

## 고정 계약

- 미국 자산: 확정된 미국 정규장 일봉 `t`까지만 사용해 `t+1` 거래일의 True Range를 예측합니다.
- 암호자산: 완전히 종료된 UTC 일봉 `t`까지만 사용합니다.
- 모델 계수와 전역 OOF 구간 분포는 2024년까지의 사전 지정 development 자산으로 한 번 고정합니다.
- 50%·80% 구간은 고정 모델의 최근 252개 실제 오차만 순차적으로 사용합니다.
- 방향 예측은 만들지 않습니다. 꼬리위험은 내부 실험값으로만 기록합니다.
- 앱의 `weatherScorev0.1` 공식은 변경하지 않습니다.

## 명령 순서

```text
pnpm shadow:freeze
pnpm shadow:dry-run
pnpm shadow:self-test
pnpm shadow:audit
```

`shadow:dry-run`은 공식 원장에 쓰지 않습니다. 공식 관측은 원천 일봉의 종료 시각과 최신성을 확인한 뒤 `shadow:snapshot`과 `shadow:forecast`를 실행합니다. 다음 거래일 데이터 갱신 뒤 `shadow:settle`로 정산합니다.

스냅샷 JSON은 별도 `.sha256` 체크섬과 함께 생성되며 예측 전에 일치 여부를 다시 확인합니다.

공식 기록은 미국 자산의 동기화된 최근 종가, 암호자산의 직전 완전 종료 UTC 일봉을 요구합니다. 오래된 캐시·미완성 당일봉·미래 문맥·라벨이 섞인 추론 행은 자동 거부합니다.

실제 운영일에는 `pnpm shadow:refresh`로 직전 완전 종료 UTC 날짜까지 공급자 데이터를 새로 받은 뒤 `pnpm shadow:forecast`, `pnpm shadow:register`를 순서대로 실행합니다. `register`는 결과가 나오기 전에 ATR 기준선과 날씨 사건 경계를 별도 해시 원장에 고정합니다. 미국 휴장일에는 Yahoo가 반환한 가장 최근 거래일을 허용하되 모든 미국 자산 날짜가 같아야 합니다.

공식 모델을 고정한 뒤에는 원장이 존재하는 동안 `--force` 재고정을 거부합니다. 같은 자산·날짜 예측은 덮어쓰지 않고 건너뛰므로, 미국 휴장 중에도 새 UTC 일봉이 생긴 암호자산만 계속 기록할 수 있습니다.
