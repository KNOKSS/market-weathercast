# 앱 적용 및 일일 운영

## 앱 데이터 경계

연구 원장은 앱에 직접 노출하지 않는다. `python scripts/shadow/run.py export`가 다음 항목만 `public/data/tomorrow-forecast.json`으로 내보낸다.

- 다음 일봉 예상 True Range
- 50%·80% 구간
- 자산별 최근 252회 대비 날씨 등급
- 예보 기준일과 정산 상태
- 최근 정산 결과

모델 계수, 원시 특징, 실험용 꼬리위험 확률은 공개 파일에서 제외한다.

## 앱 표시 규칙

- `시장날씨` 안에서 `오늘 관측 / 내일 예보`를 전환한다.
- S&P 500은 검증된 SPY, Nasdaq Composite는 QQQ를 대리 지표로 사용한다고 명시한다.
- 검증 대상이 아닌 SOL이나 임의 추가 종목에는 예보를 만들지 않는다.
- 날씨는 변동폭의 강도이며 상승·하락 방향이 아니다.
- 예보 JSON은 service worker 캐시를 우회해 항상 network-first로 확인한다.
- 미국 일봉은 4일, 암호자산 일봉은 1일보다 오래되면 `갱신 지연`으로 표시한다.

## 자동 운영

`.github/workflows/daily-market-forecast.yml`이 매일 00:25 UTC에 다음 순서를 실행한다.

1. 직전 완전 종료 UTC 날짜까지 원천 데이터 갱신
2. 결과가 생긴 이전 예보 정산
3. 새 종가의 공식 예보 기록
4. 결과가 나오기 전 기준선 사전 등록
5. 앱 공개 JSON 생성과 해시 감사
6. 빌드 통과 후 변경 원장과 공개 JSON만 커밋

GitHub 커밋이 Vercel 배포를 유발하므로 별도 앱 서버 없이 새 예보가 배포된다. 데이터 날짜가 맞지 않거나 원장 해시가 깨지면 작업은 커밋 전에 실패한다.
